# M8:安全加固与个人中心 设计文档

- **日期**:2026-06-18
- **状态**:已确认,待编写实现计划
- **关系**:在已完成系统(P1–M7 + 动态RBAC网关)上增加账号安全与个人中心功能。

## 1. 目标与范围

**包含(已与用户确认 A+B+C+D 全做)**
- **A. 改自己密码**:校验旧密码 → 设新密码 → 强制重新登录。
- **B. 管理员重置他人密码**:重置为固定默认 `123456`,并踢掉目标用户所有会话。
- **C. 个人资料编辑**:改自己的昵称/手机/邮箱/头像;头像只存 URL(不做文件上传)。
- **D. 改密/重置后踢会话**:改密成功 `StpUtil.logout` 当前用户;管理员重置后 `StpUtil.logout(目标)`。

**非目标(YAGNI)**
- 不做头像文件上传(只存 URL)。
- 不做忘记密码/邮箱找回(无真实邮件网关,M6 已 mock,本期不扩展)。
- 防爆破已在 M6 完成,本期不重复。

## 2. 后端接口

| 方法 路径 | 鉴权 | 说明 |
|---|---|---|
| GET `/auth/profile` | 登录 | 当前用户资料:username/nickname/phone/email/avatar |
| PUT `/auth/profile` | 登录 | 改自己资料(昵称/手机/邮箱/头像URL);手机&邮箱查重(排除自己) |
| PUT `/auth/password` | 登录 | 改自己密码:校验旧密码(BCrypt)→ 存新 → `StpUtil.logout(自己)` |
| PUT `/system/user/{id}/reset-password` | 权限 `system:user:edit` | 管理员重置:设 BCrypt(`123456`)→ `StpUtil.logout(目标)` |

- `/auth/profile`、`/auth/password` 在 `/auth/**` 下,网关仅校验登录并注入 `X-User-Id`,用户管自己,无需权限码。
- `/system/user/{id}/reset-password` 在 `/system` 下,受动态RBAC 管控(见 §4)。

## 3. 后端实现要点

- **DTO**:
  - `ChangePasswordRequest { oldPassword, newPassword }`(均 @NotBlank)。
  - `ProfileUpdateRequest { nickname, phone, email, avatar }`。
  - `ProfileVO { username, nickname, phone, email, avatar }`。
- **AuthService**(自助):
  - `getProfile()`:从 `X-User-Id`(`UserContext.getUserId()`)查用户,返回 ProfileVO。
  - `updateProfile(req)`:查重 phone/email(`!= 当前 userId` 才算冲突),更新 nickname/phone/email/avatar。
  - `changePassword(req)`:取当前用户,`passwordEncoder.matches(old, 存储)` 失败抛"原密码错误";否则 `setPassword(encode(new))` 更新;成功后 `StpUtil.logout(userId)`。
- **SysUserService**(管理员):
  - `resetPassword(id)`:查用户(不存在抛错);`setPassword(encode("123456"))` 更新;`StpUtil.logout(id)` 踢会话。
- **审计**:更新走 MyBatis-Plus 自动填充 `update_by`(X-User-Id),无需手动。
- 复用现有 `PasswordEncoder`(BCrypt)、`UserContext`、`SysUserMapper`。

## 4. 动态 RBAC 映射(B 的安全前提,关键)

网关鉴权现为动态查表(`sys_menu.api_url → perm`,经 Redis)。新接口 `/system/user/*/reset-password` **必须登记映射**,否则网关视为"仅需登录"→ 任何登录用户可重置他人密码(越权漏洞)。

- 在 `sys_menu` 加一行隐藏菜单(`visible=0`,挂用户管理 id=101 下):
  - `menu_type=F`, `perm=system:user:edit`, `api_method=PUT`, `api_url=/system/user/*/reset-password`。
  - 复用 `system:user:edit`(admin 已授权),不新增权限码。
- 通过种子 SQL 插入该行;auth-service 启动时 `ApiPermPublisher` 自动发布到 Redis(或菜单变更触发发布)→ 网关 pub/sub 即时生效。
- 效果:无 `system:user:edit` 的用户调该接口 → 网关返回 403。

> `ApiPermMatcher` 的"最具体优先"已能区分 `/system/user/*/reset-password`(2 段)与 `/system/user/*`(详情,1 段),不会误命中。

## 5. 前端

1. **个人中心页** `views/Profile.vue`,路由 `/profile`(布局内顶层,不入侧边栏)。入口在右上角用户下拉。两张卡:
   - 资料卡:用户名只读 + 昵称/手机/邮箱/头像URL 表单 → `PUT /auth/profile`。
   - 修改密码卡:旧密码 + 新密码 + 确认 → `PUT /auth/password`;成功后清 token → 跳 `/login`。
2. **用户管理页** `UserManage.vue`:操作列加「重置密码」按钮(`v-permission="'system:user:edit'"`)→ 确认 → `PUT /system/user/{id}/reset-password` → 提示"已重置为 123456"。
3. **MainLayout.vue**:用户下拉加「个人中心」(`command='profile'` → `router.push('/profile')`)。
4. **router**:加 `/profile` 子路由(布局下,需登录)。
5. 前端 API(`api/auth.js` / `api/user.js`):`profileApi`、`updateProfileApi`、`changePasswordApi`、`resetPasswordApi`。

## 6. 错误处理

统一 `Result` + 全局异常。原密码错误、手机/邮箱已占用、用户不存在 → `ServiceException`(`code:500` + 中文)。改密后前端收到成功即清 token 跳登录。

## 7. 测试策略

- **后端离线单测**:
  - `changePassword` 旧密码错误抛"原密码错误"(mock encoder.matches=false)。
  - `updateProfile` 手机号被他人占用抛冲突(mock mapper 查到 id≠当前)。
  - `resetPassword` 用户不存在抛错。
  - (StpUtil 静态调用在单测中难触发,以联调覆盖踢会话)。
- **联调脚本(经网关)**:改资料→GET 验证;改密→旧 token 调接口 401→新密码登录;无权限账号 reset→403,admin reset→成功且目标 token 失效。
- **前端浏览器**:个人中心改资料/改密(跳登录);用户管理重置密码。
